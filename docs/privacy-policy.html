Splits is a desktop application for timing interval training sessions and distributing performance reports to athletes. This policy explains what data the application handles and how it is used.

Data You Provide

Splits does not collect, transmit, or store any data on external servers. All data you work with in the application — including athlete rosters and session results — is created and managed by you and stored exclusively on your local device. This data is never sent anywhere except when you explicitly choose to email a report (see below).

Google User Data — Data Accessed

Splits includes an optional feature that lets you send performance reports to athletes via Gmail. To use this feature, you sign in with your Google account using OAuth 2.0. The application requests the following Google API scopes:

  • https://www.googleapis.com/auth/gmail.send — grants the ability to send email on your behalf through the Gmail API. Splits uses this scope exclusively to deliver performance reports to athlete email addresses that you specify.
  • openid email — grants access to your Google account email address. Splits uses this solely to display which account is signed in and to set the "From" address on outgoing emails.

Splits does not access, read, modify, delete, or store any of the following Google user data: your email inbox or sent mail, email drafts, contacts, Google Drive files, Google Calendar data, or any other Google service beyond the two scopes listed above.

Google User Data — Data Usage and Storage

The following Google user data is accessed or stored by Splits:

  • OAuth refresh token — issued by Google after you grant consent. Stored locally on your device (in the application's user data directory) as gmail_tokens.json. This allows the app to send email in future sessions without requiring you to sign in again. The token is never transmitted to any server other than Google's OAuth token endpoint (oauth2.googleapis.com) when the app silently refreshes an access token to send an email.
  • Your Google account email address — obtained from the OpenID Connect ID token at sign-in. Stored locally alongside the refresh token. Used only to display your signed-in account in the app and to populate the "From" field of outgoing emails.
  • Outgoing email content — the HTML performance report body and athlete recipient addresses you specify are transmitted directly from your device to Google's Gmail API endpoint (gmail.googleapis.com) at the moment you initiate a send. This content is not stored or logged by Splits.

No Google user data is transmitted to any server operated by Splits or its developers. All storage occurs exclusively on your local device. Emails are sent only when you explicitly initiate a send action — the app never sends email autonomously.

Google Sign-In Flow

The sign-in flow opens your browser to Google's consent screen using the Authorization Code + PKCE flow. A short-lived local HTTP server (on localhost) receives the authorization code callback. The server is shut down immediately after the code is received. The authorization code is exchanged for tokens directly with Google and is never logged or stored.

Revoking Access

You can sign out of Google at any time from within the Splits settings panel. You can also revoke the app's access to your Google account at any time by visiting myaccount.google.com/permissions.

No Analytics or Tracking

Splits does not include any analytics, crash reporting, telemetry, or tracking of any kind. The application has no network communication other than the Gmail API calls you explicitly initiate.

Children's Privacy

Splits is intended for use by coaches and instructors. The application does not knowingly collect any personal information from children.
